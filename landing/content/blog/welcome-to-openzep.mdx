Today marks a major milestone for the agent memory ecosystem. **OpenZync 1.0 is here.**

## Why OpenZync?

Agents are transforming how we interact with software, but they are fundamentally limited without durable memory. Every reset loses context, every cold start wastes tokens re-building state, and every failure requires replaying from scratch.

OpenZync solves this by providing a persistent, graph-based memory layer that agents can read from, write to, and reason over.

## What's in 1.0

- **10+ Graph Backends** — Neo4j, FalkorDB, Memgraph, Amazon Neptune, and more
- **5+ LLM Providers** — OpenAI, Anthropic, Google, local models via Ollama
- **Human-in-the-Loop** — Approval gates for any action with external side effects
- **Deterministic Workflows** — Stage-gated DAGs that are crash-safe and resumable
- **Built-in Observability** — Structured logs, latency tracking, cost estimates
