Today marks a major milestone for the agent memory ecosystem. **OpenZync is here.**

## Why OpenZync?

Agents are transforming how we interact with software, but they are fundamentally limited without durable memory. Every reset loses context, every cold start wastes tokens re-building state, and every failure requires replaying from scratch.

OpenZync solves this by providing a persistent, graph-based memory layer that agents can read from, write to, and reason over.

## What's Included

- **Graph Backends** — PostgreSQL (built-in), FalkorDB, and SurrealDB — all behind a single abstraction
- **LLM Providers** — OpenAI, Anthropic, Azure, OpenRouter, and local models via Ollama
- **Webhook Events** — Event-driven architecture with 15 event types and HMAC-signed delivery
- **Prompt Versioning** — Version-controlled templates per organization with rollback support
- **Built-in Observability** — Prometheus metrics, structured logging, Grafana dashboards
